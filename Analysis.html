01. What is Matplotlib?

🔁 Think of it like this...
Imagine you have a spreadsheet full of numbers — transactions, amounts, dates. You can read every row manually, but it would take hours and you'd probably miss the big picture.

Matplotlib is like turning that spreadsheet into a poster. Instead of staring at numbers, you see patterns — a spike in fraud at midnight, a bar chart showing which transaction type happens most, a scatter plot showing that large withdrawals at night look very different from regular ones.

Matplotlib draws those posters for you. You describe what you want in code, and it renders it as a chart.

In technical terms: Matplotlib is Python's most popular library for creating data visualisations. It can produce bar charts, line graphs, histograms, scatter plots, pie charts, heatmaps, and much more — all customisable and exportable as images.

💡 Quick explainer — what's a library?
A library is a collection of pre-written code that someone else built and packaged so you don't have to write it yourself. Instead of writing 500 lines to draw a bar chart from scratch, you just call plt.bar() and Matplotlib handles the rest. Think of it as a toolbox: you don't build the hammer, you just pick it up and use it.

Why does this matter for data work?

Raw numbers are hard to interpret. Charts make patterns visible instantly. This is important for:

  • Exploratory analysis — understanding your data before you model it
  • Communicating findings — showing stakeholders what's happening
  • Detecting anomalies — fraud, outliers, data quality issues
  • Feature engineering — in ML, you often create new columns based on patterns you first spot in a chart

03. Your First Chart — Bar Chart of Transaction Types

A bar chart answers the question: "How many times did each category appear?"

In our dataset, there are 5 types of transactions: send_money, buy_goods, paybill, withdraw, and deposit. Let's find out which type happens most often.

🔁 Real world version
Imagine you run a small shop and you want to know which product sells most. At the end of the week, you count how many units of each product you sold and draw a bar for each. The tallest bar = your best seller. That's a bar chart. We're doing exactly the same thing — but for transaction types.
🟡 Cell 3 — Count and plot transaction types
# Step 1: Count how many times each transaction type appears
# value_counts() is a pandas tool — it counts each unique value in a column
type_counts = df['transaction_type'].value_counts()

# Step 2: Create a figure — think of this as setting up your blank canvas
plt.figure(figsize=(10, 6))  # width=10 inches, height=6 inches

# Step 3: Draw the bar chart
# type_counts.index = the category names (x-axis labels)
# type_counts.values = the counts (bar heights)
plt.bar(type_counts.index, type_counts.values, color='#3b82f6')

# Step 4: Add labels so your chart can speak for itself
plt.title('Transaction Volume by Type', fontsize=14)
plt.xlabel('Transaction Type')
plt.ylabel('Number of Transactions')

# Step 5: Display the chart
plt.tight_layout()  # prevents labels from getting cut off
plt.show()
📊
Expected output: A bar chart with 5 bars.
send_money (~3,294) should be the tallest, followed by buy_goods (~2,651), paybill (~2,087), withdraw (~1,286), and deposit (~682).

Make it more readable — add value labels on top of bars

🟡 Cell 4 — Bar chart with labels on each bar
# A polished version — with numbers shown on top of each bar
plt.figure(figsize=(10, 6))

bars = plt.bar(
    type_counts.index,
    type_counts.values,
    color=['#3b82f6', '#10b981', '#f59e0b', '#ef4444', '#8b5cf6']
)

# Loop through each bar and write the count on top of it
for bar in bars:
    height = bar.get_height()  # how tall is this bar?
    plt.text(
        bar.get_x() + bar.get_width() / 2,  # x position: centre of the bar
        height + 20,                           # y position: just above the bar top
        str(int(height)),                       # the text to show: the count
        ha='center', fontsize=11
    )

plt.title('Transaction Volume by Type', fontsize=14)
plt.xlabel('Transaction Type')
plt.ylabel('Count')
plt.tight_layout()
plt.show()
What you just learned: plt.bar() draws bars. plt.title(), plt.xlabel(), plt.ylabel() add text labels. plt.show() renders the final chart. These 5 functions are the backbone of almost every chart you'll ever make.

04. Histogram — How Are Transaction Amounts Distributed?

A histogram answers: "What range of values appears most often?"

Where a bar chart counts categories (like "how many withdrawals?"), a histogram groups numbers into ranges and counts how many fall in each range. It shows you the shape of your data.

🔁 Here's the everyday version of that
Imagine you're a teacher grading 100 tests. You don't want to plot every single score — you want to know the shape: how many students scored between 0-20, 20-40, 40-60, etc. A histogram groups all scores into buckets and counts how full each bucket is. That same idea applies here: how many transactions fell in the KES 0-1000 range? KES 1000-2000? KES 5000+?
🟡 Cell 5 — Histogram of transaction amounts
# bins=50 means: split the range into 50 equal-width buckets
# The more bins, the more detail — but too many looks noisy
plt.figure(figsize=(10, 5))

plt.hist(df['amount_kes'], bins=50, color='#3b82f6', edgecolor='white')

plt.title('Distribution of Transaction Amounts (KES)')
plt.xlabel('Amount (KES)')
plt.ylabel('Number of Transactions')

# Add a vertical line showing the average amount
# This helps us see where "typical" sits relative to the full distribution
avg_amount = df['amount_kes'].mean()
plt.axvline(avg_amount, color='#f59e0b', linestyle='--', label=f'Mean: KES {avg_amount:.0f}')
plt.legend()  # shows the label next to the mean line

plt.tight_layout()
plt.show()
📉
Expected output: A right-skewed distribution.
Most transactions cluster at the lower amounts (under KES 5,000), but a long tail extends to the right toward KES 70,000. The mean line (KES ~5,450) sits well to the right of the peak — this is a sign of outliers pulling the average up.
💡 What does "right-skewed" mean?
Skewed means the data is not symmetrical — the distribution is pulled more to one side. Right-skewed (also called positively skewed) means most values are small, but a few very large values exist, pulling the "tail" to the right.

In our case: most people send under KES 5,000, but occasionally someone sends KES 50,000+. Those high-value outliers are important — in fraud detection, unusually large transactions are a red flag.

Zoom in on the most common range

🟡 Cell 6 — Filter to amounts under KES 20,000 for clarity
# Filter to just the transactions under 20,000 KES
# This removes extreme outliers so we can see the main pattern more clearly
df_filtered = df[df['amount_kes'] < 20000]

plt.figure(figsize=(10, 5))
plt.hist(df_filtered['amount_kes'], bins=60, color='#10b981', edgecolor='white')
plt.title('Transaction Amounts Under KES 20,000')
plt.xlabel('Amount (KES)')
plt.ylabel('Count')
plt.tight_layout()
plt.show()

05. Pie Chart — What Fraction of Transactions are Fraud?

A pie chart shows how a whole is divided into parts. We use it when we want to emphasise proportions — like the share of fraudulent vs normal transactions.

🚨 Don't worry — this tripped everyone up the first time
Pie charts get a bad reputation in data science because they can be misleading when there are many slices. But for a simple 2-part comparison (fraud vs normal), they work perfectly and are easy to read at a glance. Just avoid using them for more than 4-5 categories — use a bar chart instead for those cases.
🟡 Cell 7 — Pie chart of fraud vs normal transactions
# Count how many transactions are fraud (1) vs normal (0)
fraud_counts = df['is_fraud'].value_counts()

# Give them readable labels instead of 0 and 1
labels = ['Normal', 'Fraudulent']
colours = ['#3b82f6', '#ef4444']

plt.figure(figsize=(7, 7))

# explode = "pull out" a slice to highlight it
# (0, 0.1) means: don't pull Normal, pull Fraudulent out by 0.1
plt.pie(
    fraud_counts.values,
    labels=labels,
    colors=colours,
    autopct='%1.1f%%',   # show percentage on each slice
    explode=(0, 0.1),   # highlight the fraud slice
    startangle=90        # start at the top of the circle
)

plt.title('Fraud vs Normal Transactions', fontsize=14)
plt.show()
🥧
Expected output: A circle divided into two slices.
97.0% Normal (blue) and 3.0% Fraudulent (red, pulled out slightly).
Key insight: The fraud rate is only 3%. This is called a class imbalance. In ML, this creates a real problem — if you train a model on this data, it could predict "normal" for every transaction and still be 97% accurate, while catching zero fraud. Visualising this imbalance early is the first step to handling it properly.

06. Scatter Plot — Amount vs Deviation from Normal Behaviour

A scatter plot shows the relationship between two numerical columns. Each dot is one transaction, plotted by its two values. Patterns in the dots reveal relationships — and fraud often shows up as dots in unexpected places.

🟡 Cell 8 — Scatter plot: amount vs amount_deviation, coloured by fraud
# Separate fraud and normal transactions into two groups
# We'll plot them in different colours so we can compare
normal = df[df['is_fraud'] == 0]
fraud  = df[df['is_fraud'] == 1]

plt.figure(figsize=(10, 6))

# Plot normal transactions first (blue dots)
# alpha = transparency (0=invisible, 1=solid) — helps when dots overlap
plt.scatter(
    normal['amount_kes'],
    normal['amount_deviation'],
    alpha=0.3, color='#3b82f6', s=15, label='Normal'
)

# Plot fraud transactions on top (red dots, slightly larger)
plt.scatter(
    fraud['amount_kes'],
    fraud['amount_deviation'],
    alpha=0.7, color='#ef4444', s=40, label='Fraud', marker='x'
)

plt.title('Transaction Amount vs Deviation (Fraud Highlighted)')
plt.xlabel('Amount (KES)')
plt.ylabel('Amount Deviation (x times user average)')
plt.legend()
plt.tight_layout()
plt.show()
💡 Let me explain what amount_deviation means
Every user has a typical transaction amount — their average. The amount_deviation column tells you how many times larger (or smaller) this transaction is compared to that average.

An amount_deviation of 1.0 means this transaction is exactly the user's average. A value of 20 means this transaction is 20x larger than what this person normally does. That's suspicious. Fraudulent transactions tend to cluster at high deviation values — that's the pattern we're looking for in the scatter plot.

07. Line Chart — Transaction Volume Over Time

A line chart shows how something changes over time. We'll look at transaction volume month by month to spot seasonal patterns.

🟡 Cell 9 — Convert timestamp and plot monthly volume
# Right now, timestamp is just text. We need to convert it to a real date type
# pd.to_datetime() tells pandas: "treat this column as actual dates"
df['timestamp'] = pd.to_datetime(df['timestamp'])

# Extract just the month from each date
# .dt is the "datetime accessor" — it lets you pull parts of a date
df['month'] = df['timestamp'].dt.to_period('M')  # 'M' = month

# Count how many transactions happened in each month
monthly_volume = df.groupby('month').size()

plt.figure(figsize=(12, 5))

# Convert month labels to strings so matplotlib can display them properly
plt.plot(
    monthly_volume.index.astype(str),
    monthly_volume.values,
    color='#3b82f6',
    linewidth=2,
    marker='o',    # adds a dot at each data point
    markersize=6
)

plt.title('Monthly Transaction Volume (2024)')
plt.xlabel('Month')
plt.ylabel('Number of Transactions')
plt.xticks(rotation=45)  # rotate x labels so they don't overlap
plt.grid(axis='y', alpha=0.3)  # adds faint horizontal gridlines
plt.tight_layout()
plt.show()

Compare fraud vs normal over time on the same chart

🟡 Cell 10 — Two lines on one chart
# Count monthly normal transactions
monthly_normal = df[df['is_fraud']==0].groupby('month').size()

# Count monthly fraud transactions
monthly_fraud  = df[df['is_fraud']==1].groupby('month').size()

fig, ax1 = plt.subplots(figsize=(12, 5))

# Left y-axis: normal transaction counts (blue)
ax1.plot(monthly_normal.index.astype(str), monthly_normal.values,
        color='#3b82f6', linewidth=2, marker='o', label='Normal')
ax1.set_ylabel('Normal Transactions', color='#3b82f6')

# Right y-axis: fraud counts (red) — different scale
ax2 = ax1.twinx()  # twinx() creates a second y-axis sharing the same x-axis
ax2.plot(monthly_fraud.index.astype(str), monthly_fraud.values,
        color='#ef4444', linewidth=2, marker='s', label='Fraud')
ax2.set_ylabel('Fraudulent Transactions', color='#ef4444')

plt.title('Normal vs Fraudulent Transactions by Month')
ax1.set_xlabel('Month')
plt.xticks(rotation=45)
fig.legend(loc='upper right')
plt.tight_layout()
plt.show()

08. Visualising Fraud Patterns — When and How Fraud Happens

Now we get to the most interesting part. Fraud doesn't happen randomly — it has patterns. Let's use charts to find them.

Pattern 1 — What hour of the day does fraud happen?

🟡 Cell 11 — Fraud by hour of day
# Group by hour, count fraud cases in each hour
fraud_by_hour = df[df['is_fraud']==1].groupby('hour_of_day').size()

plt.figure(figsize=(12, 5))

plt.bar(fraud_by_hour.index, fraud_by_hour.values, color='#ef4444', alpha=0.85)

# Highlight the night hours (0-5am) with a shaded region
plt.axvspan(0, 5.5, alpha=0.1, color='yellow', label='Night (midnight–5am)')

plt.title('Fraud Transactions by Hour of Day')
plt.xlabel('Hour (0 = midnight, 12 = noon, 23 = 11pm)')
plt.ylabel('Number of Fraud Cases')
plt.xticks(range(0, 24))  # show all 24 hours on the x-axis
plt.legend()
plt.tight_layout()
plt.show()

Pattern 2 — Which transaction type is most targeted?

🟡 Cell 12 — Fraud rate by transaction type
# Calculate the fraud rate for each transaction type
# We want: for each type, what % of transactions are fraud?
fraud_rate = df.groupby('transaction_type')['is_fraud'].mean() * 100
fraud_rate = fraud_rate.sort_values(ascending=False)

plt.figure(figsize=(10, 5))

bars = plt.bar(fraud_rate.index, fraud_rate.values, color='#ef4444')

# Add a dotted line showing the overall average fraud rate
plt.axhline(y=df['is_fraud'].mean() * 100,
           color='#f59e0b', linestyle='--', label='Overall fraud rate (3%)')

plt.title('Fraud Rate (%) by Transaction Type')
plt.xlabel('Transaction Type')
plt.ylabel('Fraud Rate (%)')
plt.legend()
plt.tight_layout()
plt.show()
Key insight: Some transaction types will have fraud rates above the 3% average. That's a signal a feature engineer would use — "is this a high-risk transaction type?" becomes a useful column when building a fraud detection model.

09. Subplots — Multiple Charts in One Figure

So far we've made one chart at a time. In real projects, you often want a dashboard view — multiple charts displayed together. That's what plt.subplots() is for.

🔁 In plain life terms...
Think of a car dashboard. You don't have one giant speedometer taking up the whole panel — you have speed, RPM, fuel level, and temperature all visible at once in separate sections. plt.subplots() lets you build that same kind of multi-panel display for your data.
🟡 Cell 13 — A 2x2 dashboard of charts
# fig = the overall figure (the whole canvas)
# axes = a grid of chart positions (2 rows, 2 columns = 4 charts)
fig, axes = plt.subplots(2, 2, figsize=(14, 10))

# ── Chart 1 (top-left): Transaction type bar chart ──
type_counts = df['transaction_type'].value_counts()
axes[0, 0].bar(type_counts.index, type_counts.values, color='#3b82f6')
axes[0, 0].set_title('Transaction Types')
axes[0, 0].set_ylabel('Count')

# ── Chart 2 (top-right): Amount histogram ──
axes[0, 1].hist(df['amount_kes'], bins=40, color='#10b981', edgecolor='white')
axes[0, 1].set_title('Amount Distribution')
axes[0, 1].set_xlabel('KES')

# ── Chart 3 (bottom-left): Fraud by hour ──
fraud_by_hour = df[df['is_fraud']==1].groupby('hour_of_day').size()
axes[1, 0].bar(fraud_by_hour.index, fraud_by_hour.values, color='#ef4444')
axes[1, 0].set_title('Fraud by Hour of Day')
axes[1, 0].set_xlabel('Hour')

# ── Chart 4 (bottom-right): Fraud rate by type ──
fraud_rate = df.groupby('transaction_type')['is_fraud'].mean() * 100
axes[1, 1].bar(fraud_rate.index, fraud_rate.values, color='#8b5cf6')
axes[1, 1].set_title('Fraud Rate by Type (%)')
axes[1, 1].set_ylabel('Fraud %')

# Add a main title over all 4 charts
fig.suptitle('M-Pesa Transaction Analysis Dashboard', fontsize=16, y=1.02)

# Automatically adjust spacing so charts don't overlap
plt.tight_layout()
plt.show()
💡 Here's what's going on under the hood — axes[row, col]
When you create a 2x2 grid with plt.subplots(2, 2), Python creates a grid of 4 "axes" objects — think of each one as a separate mini-chart container. You access each container using row and column numbers starting from 0:
  • axes[0, 0] = top-left chart
  • axes[0, 1] = top-right chart
  • axes[1, 0] = bottom-left chart
  • axes[1, 1] = bottom-right chart
Instead of calling plt.bar(), you call axes[0, 0].bar() — same function, but you're saying "draw this in that specific container".

10. Saving Your Charts as Image Files

Once your chart looks good, you'll want to save it. Maybe for a report, a presentation, or a portfolio.

🟡 Save a chart before plt.show()

plt.figure(figsize=(10, 5))

plt.title('Transaction Types')

# dpi = dots per inch — higher = better quality
# bbox_inches='tight' = don't cut off labels
plt.savefig('transaction_types.png', dpi=150, bbox_inches='tight')

plt.show()

print("Chart saved!")

12. How It All Fits Together

Let's zoom out and look at what you've actually learned today — not just the syntax, but the thinking behind it.

Chart Type Question it answers Key function
Bar chart How many in each category? plt.bar()
Histogram What's the shape of this number? plt.hist()
Pie chart What fraction of the whole? plt.pie()
Scatter plot Are two columns related? plt.scatter()
Line chart How does this change over time? plt.plot()
Subplots Show multiple charts at once plt.subplots()
The bigger picture: Every chart you made today is part of what data scientists call Exploratory Data Analysis (EDA). EDA is the first step before any ML model. You visualise the data to understand its shape, find patterns, spot outliers, and identify which features might be useful for prediction. The fraud patterns you visualised today (night hours, large amounts, high deviation) are exactly the kinds of signals a machine learning model would be trained to detect automatically.
